# Section 8.4 - Finalize and Test the Agentic Workflow

The final configuration steps must be completed before the Agentic Workflow can be used.

## Review Workflow Configuration

1. Continue through the remaining workflow configuration sections:

   - Define security controls
   - Add triggers
   - Select channels and status

   ![](https://raw.githubusercontent.com/kerianngarcia/put-ai-to-work-custom-lab/main/.gitbook/assets/recovered-0786cdd115804b.png)

## Configure Security Controls

2. Open **Define Security Controls**.

   ![](https://raw.githubusercontent.com/kerianngarcia/put-ai-to-work-custom-lab/main/.gitbook/assets/recovered-8222a0b96c219e.png)

3. Configure user access using the following value.

   | Setting | Value |
   |---|---|
   | Define user access | Allow if roles: `itil` |

4. Configure data access.

   | Setting | Value |
   |---|---|
   | User identity type | AI user |
   | AI user | itsm.aia.worker |

   ![](https://raw.githubusercontent.com/kerianngarcia/put-ai-to-work-custom-lab/main/.gitbook/assets/recovered-384bb4357ce54d.png)

## Configure the Trigger

5. Activate the out-of-the-box copy trigger.

6. Set **Trigger is ON**.

   ![](https://raw.githubusercontent.com/kerianngarcia/put-ai-to-work-custom-lab/main/.gitbook/assets/recovered-29806c329d8546.png)

## Configure Channels and Status

7. Open **Select Channels and Status**.

8. Verify that **Engage via the Now Assist Panel** is enabled.

   ![](https://raw.githubusercontent.com/kerianngarcia/put-ai-to-work-custom-lab/main/.gitbook/assets/recovered-285789d0f04853.png)

## Test the Agentic Workflow

9. Click **Save and Test**.

10. Test the workflow using one of the following options:

   - Create a new incident.
   - Use an existing incident in testing mode.
   - Consider testing with higher education scenarios such as:
     - Student cannot access Canvas
      - Faculty gradebook synchronization issue
      - Researcher unable to access secure storage
      - Staff member unable to access Workday
      - Classroom technology outage

11. Observe how the workflow identifies the impacted service and routes the issue to the appropriate support team.

    ![](https://raw.githubusercontent.com/kerianngarcia/put-ai-to-work-custom-lab/main/.gitbook/assets/recovered-19a93a5b58e23c.png)

12. Confirm that:

    - Incident fields are updated automatically.
    - Assignment information is populated.
    - Work notes are generated by the AI Agents.
    - The workflow executes successfully from end to end.

## Completion

Congratulations. You completed this lab section and created an Agentic Workflow that includes a custom AI Agent.
